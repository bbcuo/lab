☕ Java 基础语法

Java 中有 8 种基本数据类型（primitive types），分为四类：整型、浮点型、字符型和布尔型。变量必须先声明后使用，运算符包括算术、关系、逻辑、赋值等。
📄 数据类型与变量
public class BasicDemo {
    public static void main(String[] args) {
        // 整型
        byte b = 127;              // 1字节，-128 ~ 127
        short s = 32767;           // 2字节
        int i = 2147483647;        // 4字节（最常用）
        long l = 9223372036854775807L; // 8字节，加 L

        // 浮点型
        float f = 3.14f;           // 4字节，加 f
        double d = 3.1415926535;   // 8字节（默认）

        // 字符型
        char c = 'A';              // 2字节，Unicode 编码

        // 布尔型
        boolean flag = true;       // 只有 true / false

        // 自动类型转换：byte → short → int → long → float → double
        double result = i + d;     // int 自动提升为 double
    }
}
📄 运算符
int a = 10, b = 3;
System.out.println(a + b);   // 13  加法
System.out.println(a - b);   // 7   减法
System.out.println(a * b);   // 30  乘法
System.out.println(a / b);   // 3   整数除法（取整）
System.out.println(a % b);   // 1   取余

// 关系运算符：== != > < >= <=
// 逻辑运算符：&&（与） ||（或） !（非）
boolean result = (a > 5) && (b < 5);  // true

// 三元运算符
String msg = (a > b) ? "a 更大" : "b 更大";

☕ Java 面向对象

面向对象编程（OOP）的三大特性：封装（用 private 隐藏细节）、继承（用 extends 复用代码）、多态（父类引用指向子类对象）。
📄 封装与继承
// 父类
public class Animal {
    private String name;          // private：封装，外部不可直接访问

    public Animal(String name) {  // 构造方法
        this.name = name;
    }

    public String getName() {     // getter：提供受控访问
        return name;
    }

    public void makeSound() {
        System.out.println("动物发出声音");
    }
}

// 子类继承父类
public class Dog extends Animal {
    public Dog(String name) {
        super(name);              // 调用父类构造方法
    }

    @Override                     // 重写父类方法
    public void makeSound() {
        System.out.println(getName() + "：汪汪汪！");
    }
}
📄 多态
public class Test {
    public static void main(String[] args) {
        // 多态：父类引用指向子类对象
        Animal a1 = new Dog("旺财");
        Animal a2 = new Cat("咪咪");

        a1.makeSound();  // 旺财：汪汪汪！
        a2.makeSound();  // 咪咪：喵喵喵！

        // instanceof 判断实际类型
        if (a1 instanceof Dog) {
            Dog d = (Dog) a1;     // 向下转型
        }
    }
}

☕ Java 集合框架

Java 集合框架主要分为三大类：List（有序可重复）、Set（无序不可重复）、Map（键值对）。常用实现类：ArrayList、HashSet、HashMap。
📄 ArrayList 与 HashSet
import java.util.*;

public class CollectionDemo {
    public static void main(String[] args) {
        // ArrayList：有序，可重复，有索引
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("Java");          // 允许重复
        System.out.println(list);  // [Java, Python, Java]
        System.out.println(list.get(0)); // Java

        // HashSet：无序，不可重复
        Set<String> set = new HashSet<>();
        set.add("Java");
        set.add("Python");
        set.add("Java");           // 重复，不会添加
        System.out.println(set);   // [Java, Python]（顺序不固定）
    }
}
📄 HashMap 键值对
// HashMap：键值对，键不可重复
Map<String, Integer> map = new HashMap<>();
map.put("张三", 90);
map.put("李四", 85);
map.put("张三", 95);              // 覆盖旧值

System.out.println(map.get("张三")); // 95

// 遍历 Map
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}

🎨 CSS Flexbox 布局

Flexbox（弹性盒子）是一维布局模型，通过 display: flex 激活。核心属性：justify-content（主轴对齐）、align-items（交叉轴对齐）、flex-direction（方向）。
📄 Flexbox 常用布局
/* ===== 容器属性 ===== */
.container {
    display: flex;
    justify-content: center;      /* 主轴：flex-start | center | flex-end | space-between | space-around */
    align-items: center;          /* 交叉轴：flex-start | center | flex-end | stretch */
    flex-direction: row;          /* 方向：row | row-reverse | column | column-reverse */
    flex-wrap: wrap;              /* 换行：nowrap | wrap */
    gap: 20px;                    /* 子元素间距 */
}

/* ===== 子元素属性 ===== */
.item {
    flex: 1;                      /* 等分剩余空间 */
    /* 等价于 flex-grow: 1; flex-shrink: 1; flex-basis: 0; */
}

.item:nth-child(2) {
    flex: 2;                      /* 占 2 份 */
}

🎨 CSS 动画与过渡

transition（过渡）用于属性值变化时的平滑动画；animation + @keyframes（动画）用于更复杂的、可循环的动画效果。
📄 transition 过渡
/* transition: 属性 时长 缓动函数 延迟 */
.button {
    background: #667eea;
    transition: background 0.3s ease, transform 0.2s;
}
.button:hover {
    background: #764ba2;
    transform: scale(1.05);       /* 放大 5% */
}
📄 animation + @keyframes
/* 定义关键帧 */
@keyframes slideIn {
    from { opacity: 0; transform: translateY(-20px); }
    to   { opacity: 1; transform: translateY(0); }
}

/* 使用动画 */
.card {
    animation: slideIn 0.5s ease forwards;
    /* 简写：名称 时长 缓动 次数 方向 填充模式 */
}

/* 无限旋转 */
@keyframes spin {
    to { transform: rotate(360deg); }
}
.loader {
    animation: spin 1s linear infinite;
}

🎨 CSS 响应式设计

响应式设计让网页在不同设备上都能良好显示。核心工具：@media 媒体查询、相对单位（%、vw/vh、rem）、viewport 元标签。
📄 @media 媒体查询
/* 移动端优先：基础样式针对小屏 */
.container { padding: 16px; font-size: 14px; }

/* 平板（≥768px） */
@media (min-width: 768px) {
    .container { padding: 24px; font-size: 16px; }
}

/* 桌面端（≥1024px） */
@media (min-width: 1024px) {
    .container { max-width: 1100px; margin: 0 auto; }
}

/* 打印样式 */
@media print {
    nav, footer { display: none; }
}
📄 viewport 与相对单位
<!-- head 中必须添加 viewport 元标签 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

/* 相对单位 */
.hero { height: 100vh; }         /* 100% 视口高度 */
.card { width: 90%; }            /* 父元素宽度的 90% */
.title { font-size: 2rem; }      /* 根元素字体大小的 2 倍 */

🔧 Git 基本命令

Git 是分布式版本控制系统。基本工作流：工作区 → 暂存区 → 本地仓库 → 远程仓库。
💻 Git 基本工作流
# 初始化仓库
git init

# 配置用户信息
git config user.name "bbcuo"
git config user.email "your-email@example.com"

# 查看文件状态
git status

# 添加到暂存区
git add .                    # 添加所有文件
git add index.html           # 添加指定文件

# 提交到本地仓库
git commit -m "提交信息"

# 查看提交历史
git log --oneline            # 简洁模式
git log --graph --all        # 图形化分支历史

# 撤销操作
git restore 文件名            # 撤销工作区修改
git reset HEAD 文件名         # 取消暂存
git reset --soft HEAD~1      # 撤销最近一次 commit（保留修改）

🔧 Git 分支管理

分支是 Git 的核心功能，允许并行开发互不干扰。常用操作：创建分支、切换分支、合并分支、解决冲突。
💻 分支操作
# 查看分支
git branch                   # 本地分支
git branch -a                # 所有分支（含远程）

# 创建与切换分支
git branch feature-login     # 创建分支
git checkout feature-login   # 切换分支
git checkout -b feature-pay  # 创建并切换（一步到位）

# 合并分支
git checkout main            # 先切回目标分支
git merge feature-login      # 合并

# 删除分支
git branch -d feature-login  # 删除已合并的分支
git branch -D feature-login  # 强制删除

# 变基（保持提交历史线性）
git rebase main              # 把当前分支变基到 main

🔧 Git 远程协作

远程仓库用于团队协作和代码备份。核心操作：clone、push、pull、fetch。常用平台：GitHub、Gitee、GitLab。
💻 远程仓库操作
# 克隆远程仓库
git clone https://github.com/bbcuo/lab.git

# 关联远程仓库
git remote add origin https://github.com/bbcuo/lab.git
git remote -v                # 查看远程地址

# 推送
git push -u origin main      # 首次推送（关联上游）
git push                     # 之后直接推送

# 拉取
git pull origin main         # 拉取并合并（fetch + merge）
git fetch origin             # 只拉取，不合并

# 解决冲突
# 当 pull 出现冲突时：
# 1. 手动编辑冲突文件
# 2. git add .
# 3. git commit -m "解决冲突"
# 4. git push

🗄️ MySQL CRUD 操作

CRUD 是数据库四种基本操作：Create（INSERT）、Read（SELECT）、Update（UPDATE）、Delete（DELETE）。
💻 CRUD 基本操作
-- 创建数据库
CREATE DATABASE mydb;
USE mydb;

-- 创建表
CREATE TABLE student (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    age INT,
    score DECIMAL(5, 2)
);

-- Create：插入数据
INSERT INTO student (name, age, score)
VALUES ('张三', 20, 88.5),
       ('李四', 21, 92.0);

-- Read：查询数据
SELECT * FROM student;
SELECT name, score FROM student WHERE score > 90;
SELECT * FROM student ORDER BY score DESC;
SELECT COUNT(*), AVG(score) FROM student;  -- 聚合函数
SELECT * FROM student LIMIT 5 OFFSET 0;    -- 分页

-- Update：更新数据
UPDATE student SET score = 95 WHERE name = '张三';

-- Delete：删除数据
DELETE FROM student WHERE id = 1;

🗄️ MySQL 多表查询

多表查询通过 JOIN 连接多个表。常用类型：INNER JOIN（取交集）、LEFT JOIN（左表全保留）、RIGHT JOIN（右表全保留）。
💻 JOIN 多表查询
-- 学生表
CREATE TABLE student (id INT PRIMARY KEY, name VARCHAR(20), class_id INT);
-- 班级表
CREATE TABLE class (id INT PRIMARY KEY, class_name VARCHAR(20));

INSERT INTO student VALUES (1, '张三', 1), (2, '李四', 2), (3, '王五', NULL);
INSERT INTO class VALUES (1, '一班'), (2, '二班');

-- INNER JOIN：只返回匹配的行
SELECT s.name, c.class_name
FROM student s
INNER JOIN class c ON s.class_id = c.id;
-- 结果：张三 | 一班，李四 | 二班（王五没有班级，不出现）

-- LEFT JOIN：左表全部保留
SELECT s.name, c.class_name
FROM student s
LEFT JOIN class c ON s.class_id = c.id;
-- 结果：张三 | 一班，李四 | 二班，王五 | NULL

🗄️ MySQL 索引与优化

索引就像书的目录，能大幅提升查询速度。但索引也会占用空间并降低写入速度。常用：普通索引、唯一索引、复合索引。
💻 索引操作
-- 创建索引
CREATE INDEX idx_name ON student(name);           -- 普通索引
CREATE UNIQUE INDEX idx_email ON user(email);     -- 唯一索引
CREATE INDEX idx_name_age ON student(name, age);  -- 复合索引

-- 查看索引
SHOW INDEX FROM student;

-- 删除索引
DROP INDEX idx_name ON student;

-- 查看执行计划（判断是否使用索引）
EXPLAIN SELECT * FROM student WHERE name = '张三';
-- key 列：NULL 表示未使用索引，有值表示使用了索引
📄 索引优化原则
-- ✅ 适合建索引的场景
-- 1. WHERE 子句经常使用的列
-- 2. ORDER BY 排序的列
-- 3. JOIN 关联的列

-- ❌ 不适合建索引的场景
-- 1. 表数据量很小
-- 2. 频繁更新的列
-- 3. 区分度低的列（如性别：男/女）

-- 最左前缀原则（复合索引）
-- 索引 (name, age) 可以用在：
-- WHERE name = '张三'              ✅
-- WHERE name = '张三' AND age = 20 ✅
-- WHERE age = 20                   ❌ 跳过了最左列
